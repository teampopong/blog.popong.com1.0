한글 이미지 문서에서 오픈소스 OCR 이용해서 텍스트 추출하기

Posted by @echojuliett
Aug 7th, 2013

오픈 데이터(open data)의 수준은 다음의 일곱 가지 조건으로 평가됩니다.1

  1. 데이터가 축적되고 있는지
  2. 데이터가 실시간으로 축적되고 있는지
  3. 데이터가 모두에게 공개되고 있는지
  4. 데이터가 전자화되었는지
  5. 데이터가 오픈 라이센싱 되었는지
  6. 데이터가 기계적으로 처리 가능(machine readable)한지
  7. 데이터 전체를 한꺼번에 다운로드(batch downloading) 받을 수 있는지

위의 일곱가지 항목중 현재 대한민국 국회는 1-4번을 실행하고 있습니다.2

오늘은 이 중 이미 실행되고 있는 4번과 실행되고 있지 않은 6번에 걸쳐 이야기를 해보도록 하겠습니다. 먼저, 국회의 의안정보시스템은 각 의안의 원문 및 회의록을 PDF 또는 HWP 형태로 제공하고 있으며, 문서를 전자화하는데 성공했습니다. 그러나 PDf나 HWP는 기계적으로 처리가 가능(machine readable)하지 않아 오픈 데이터(open data)의 기준을 충족하지 못합니다.

기계적으로 처리 가능한 데이터(Machine readable)란? …

한 가지 덧붙이자면, 의회 개방성 선언에 따르면 국회에서 개방하는 문서는 기계적으로 처리가 가능할 뿐더러 비상업적 도구를 이용한 개방된 형태(open format)을 이용해야한다고 합니다.

따라서 HWP나 PDF보다는 HTML, XMl, CSV 등의

,hwp,pdf3,HTML,XML,CSV 기계적으로 처리 가능 (machine readable),,,o,o,o 비상업적 포맷,,o,o,o,o

엊그제 미국의 법률 데이터가 처음으로 정부에 의해 open format으로 제공되었다며 많은 분들이 기뻐하엿습니다

The U.S. Code arrives in XML - a fully useable, robust and open format - for the first time: http://t.co/tNFx6LPKNB #opendata

— Sunlight Foundation (@SunFoundation) August 4, 2013

팀포퐁에서는 이 의안들의

  1. HTML 제공
  2. 용어 풀이
  3. 텍스트 분석

를 위해 이 의안들을 텍스트로 변환하고 있습니다.

PDF를 텍스트 내지 HTML로 변환하는 도구는 많아서 (예1, 예2, 예3)

한국어 오픈소스 OCR 라이브러리 두 가지를 살펴보았습니다.

  1. Tesseract
  2. i2ocr
